# Logout Endpoint Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ajouter `POST /auth/logout` qui invalide le token Bearer actuel en générant un nouveau token uid2 en base.

**Architecture:** Rotation de token — `logout(token)` trouve l'utilisateur, génère un nouveau `uid2(64)`, le sauvegarde en DB. L'ancien token devient immédiatement invalide. Pattern cohérent avec `changePassword`.

**Tech Stack:** Express 5, TypeScript strict, Mongoose, uid2

## Global Constraints

- Layered architecture stricte : routes → controllers → services → models
- Controllers : pas d'accès DB direct, validation via `req.token` (fourni par `isAuthenticated`)
- Services : pas de `req`/`res`, throw `HttpError` sur erreur
- `/// <reference path="../types/express.d.ts" />` requis dans tout fichier accédant à `req.token`
- Réponses d'erreur gérées par le middleware global `src/middlewares/error.ts`

- [ ] **Step 2 : Ajouter la fonction `logout` en fin de fichier**

Ajouter après la fonction `changePassword` :

```ts
export const logout = async (token: string): Promise<void> => {
  const user = await User.findOne({ token });
  if (!user) throw new HttpError("Unauthorized", 401);

  const newToken = uid2(64);
  await User.findOneAndUpdate({ token }, { token: newToken });
};
```

Dans `src/controllers/authController.ts`, ajouter après `passwordController` :

export const logoutController = async (req: Request, res: Response) => {
  const { token } = req;
  if (!token) throw new HttpError("Unauthorized", 401);

  await authService.logout(token);
  res.status(200).json({ message: "Logged out successfully" });
};
